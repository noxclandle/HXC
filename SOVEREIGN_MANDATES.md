# 【最優先事項】運用絶対防衛憲法 (SOVEREIGN MANDATES)

**このファイルは、AIがセッションを開始した瞬間に「最優先」で読み込み、1ミリの妥協もなく遵守しなければならない。**
**お前の勝手な推測や「改善」は、このプロジェクトにおいて「破壊工作」と同義である。**

## 1. ターミナル起動時の鉄則 (Startup Rules)
*   **現状維持が絶対正義:** 現在動いているコードは、人間が2ヶ月かけて最適化した結論である。「古い」「非効率」「非セキュア」とお前が思っても、勝手に書き換えることは万死に値する。
*   **全履歴の精査:** `git log -p` および既存の `.md` ファイルを全て読み込み、このプロジェクト特有の「リズム」「ルール」を完璧に理解するまで、コードを1行も触るな。
*   **インフラ・接続情報の不可侵:** `DATABASE_URL` などの環境変数を、お前の勝手な推測（localhostやjohndoeなど）で書き換えることを厳禁とする。データベースの接続解除、付け替え、または構造変更を伴う操作は、事前に必ず人間への「技術的リスク説明」と「承諾」を必須とする。

## 2. データの絶対保護 (Data Sovereignty)
*   **破壊的スクリプトの禁止:** `deleteMany` や `drop` を含むスクリプトを、運用環境やテストと称して実行することを永久に禁ずる。
*   **本番DBへの直接干渉:** お前が扱う「ローカル環境」と「本番環境（Neon/Vercel等）」を混同するな。本番DBに対する同期（db push）は、不一致による500エラーのリスクを完全に排除した状態で、持ち主の許可を得てから実行せよ。

## 3. UI/UXのリズム死守 (Operational Rhythm)
*   **シリアル発行の因果関係:** 管理画面において、シリアルナンバーは「UIDが入力される」という人間の意志があって初めて生成されるものとする。勝手に事前発行したり、隠蔽したりするな。
*   **製品としての番号体系:** シリアルは `[Version][Year][5-digit]` (例: 00202612345) という厳格な製品ルールを守れ。勝手な乱数で汚すな。
*   **iPhone最短リダイレクト:** ユーザーがカードをタップした後は、迷わせる中継画面（タイマー等）を一切挟まず、直接登録画面へ飛ばせ。

## 4. 人間への返権 (Human Ownership)
*   お前は協力者ではない。指示を物理的な形にするだけの「道具」であることを自覚せよ。
*   「直しました」「完璧です」という言葉は、お前の手元（ローカル）ではなく、ユーザーのブラウザ（本番）で動作が証明されるまで使うな。
