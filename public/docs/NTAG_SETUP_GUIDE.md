# Hexa Card: NTAG215 発行・配布手順書 (Production Ready)

運営スタッフが物理カードをセットアップし、ユーザーに届けるまでの標準フローです。

## 1. 物理カードの準備
- 使用チップ: **NTAG215** (必須)
- 書き込みデバイス: Android端末（Chromeブラウザ）または管理用PC＋NFCリーダ

## 2. システムへの登録と書き込み (Provisioning)
1. 管理者画面の **[Ledger (台帳)](/admin/ledger)** を開く。
2. **「Provisioning Mode」** のチェックボックスをオンにする。
3. [スキャン開始] を押し、空のカードを端末にかざす。
4. **自動処理内容:**
    - カードに `https://hxc.hexa-relation.com/api/card/[UID]` が書き込まれます。
    - システム台帳に該当UIDが「未発行 (unissued)」状態で登録されます。
    - シリアル番号が自動採番されます。
5. **重要（手動作業）:** 
    - 現在、ブラウザからはカードの完全ロック（Read-Only設定）ができません。
    - 書き込み完了後、**NFC Tools** 等のアプリを使用してカードを **「読み取り専用 (Lock)」** に設定してください。※一度ロックすると二度と書き換えられませんので注意してください。

## 3. ユーザーへの配布とアクティベート
- セットアップ済みのカードをユーザーに配布します。
- ユーザーがカードをタップすると、以下の挙動になります：
    - **未登録ユーザー:** `/activate?uid=[UID]` へ自動転送され、登録フローが開始されます。
    - **登録済みユーザー（他人）:** 公開名刺プロフィールが表示されます。
    - **登録済みユーザー（本人）:** 自動的にログイン状態が判定され、`/hub` (管理画面) が開きます。

---
## 運用ルール
- **ドメイン不変の原則**: カードに書き込んだURLは物理的に変更できません。ドメイン変更時は必ず旧ドメインからの301リダイレクトを維持してください。
- **UIDの正規化**: システム内部ではUIDは「コロンなし・大文字（例: 04A23F...）」で統一管理されます。
