# Hexa Relation (HXC) 運用・発送マニュアル
**— 物理資産の生成からユーザーへの譲渡まで —**

本マニュアルは、HXCシステムの管理者が「物理カード（Asset）」を安全に準備し、注文者に発送するための完全な手順を記したものです。専門知識がなくても、この手順に従えば「絶対に」ミスなく運用できるように設計されています。

---

## 1. 事前準備（必要なもの）

運用を開始する前に、以下のものが手元にあることを確認してください。

1.  **物理カード**: NTAG215チップを搭載したカード（新品）。
2.  **書き込み用スマホ**: NFC機能付きのスマホ（iPhoneを推奨）。
3.  **書き込みアプリ**: 「NFC Tools」（App Store / Google Playで無料配布されているもの）をインストールしておいてください。
4.  **管理者アカウント**: システムの「カード台帳（Registry）」にアクセスできる権限。

---

## 2. 出荷ワークフロー（5つのステップ）

### 【STEP 1】 注文（オーダー）の確認
1.  管理者ダッシュボードの「カード台帳」を開くか、直接 `/admin/registry` にアクセスします。
2.  一番上の **「Pending Shipments（未発送の注文）」** セクションを見ます。
3.  ここに表示されているユーザーが、次にカードを届けるべき相手です。

### 【STEP 2】 新品カードの「指紋(UID)」を登録する
1.  手元の新品カードをスマホの背面に当て、NFC Toolsアプリで「読み取り（Read）」を行います。
2.  **「Serial Number（またはUID）」** と表示される14桁程度の英数字（例: `04:A2:3B...`）を確認します。
3.  Registry画面の **「Asset Provisioning」** 欄にある「Card UID」入力ボックスに、その英数字を入力します。
    *   *※コロン(:)やスペースは自動で消去されるので、そのまま打って構いません。*
4.  「Register」ボタンを押します。これでカードがシステムに認識されました。

### 【STEP 3】 注文とカードを紐付ける（出荷準備）
1.  STEP 1で見つけた注文の横にある **「Assign Card」** ボタンを押します。
2.  表示されるリストから、今登録したばかりのUID（一番上に来ます）を選択します。
3.  **「Select & Ship」** を押すと、その注文は「発送済み（SHIPPED）」に切り替わり、カードと顧客がシステム上で紐付きます。

### 【STEP 4】 カードへの「命（URL）」の書き込み
1.  Registry画面下の「Master Ledger（中央台帳）」一覧から、今紐付けたカードを探します。
2.  右側の **「Copy Provisioning URL」** ボタンを押します。
    *   *※「URLがコピーされました」と表示されます。*
3.  スマホの **NFC Toolsアプリ** を開き、以下の操作をします：
    *   `Write (書き込み)` → `Add a record (レコード追加)` → `URL / URI` を選択。
    *   入力欄に、先ほどコピーしたURLをペースト（貼り付け）して `OK`。
    *   `Write` ボタンを押し、カードをスマホに当てます。「Approach your NFC tag」が「Write complete!」になれば成功です。

### 【STEP 5】 物理的ロック（二度と書き換えられないようにする）
1.  **※重要※**: これを忘れると、第三者にURLを書き換えられるリスクがあります。
2.  NFC Toolsアプリの `Other (その他)` → **`Lock tag (タグをロック)`** を選択します。
3.  カードをスマホに当てます。「Lock complete!」と出れば、そのカードは「世界に一枚だけの装備品」として完成です。
4.  カードを封筒に入れ、顧客へ発送してください。

---

## 3. ユーザーが受け取った後の流れ（確認用）

発送したカードをユーザーがスマホでタップすると、以下の順序で自動処理されます。

1.  **自動認証**: カード内の特殊なURLが読み取られ、システムが「これは本物の未登録カードだ」と判定します。
2.  **身元隠蔽**: ブラウザの履歴に秘密のコードが残らないよう、一瞬で「登録専用URL」にすり替わります。
3.  **名前の登録**: ユーザーが自分の名前とパスワードを決めます。
4.  **固定**: 登録が完了した瞬間、そのカードは「そのユーザー専用」になり、二度と他の人は登録できなくなります。

---

## 4. トラブルシューティング

| 症状 | 原因と対策 |
| :--- | :--- |
| **UIDを登録しようとするとエラーが出る** | すでにそのカードが登録されている可能性があります。台帳一覧でUIDを検索してください。 |
| **ユーザーから「読み取ってもエラーになる」と言われた** | 書き込んだURLが間違っているか、STEP 3の紐付けが完了していない可能性があります。台帳でカードのステータスが「shipped」になっているか確認してください。 |
| **間違ったUIDを注文に紐付けてしまった** | 現在、管理画面から一度紐付けたものを解除するにはエンジニアの操作が必要です。慎重に行ってください。 |
